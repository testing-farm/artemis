== IBM Cloud VPC

=== Authentication

The IBM Cloud VPC driver authenticates using an API key:

.Specification
[source,yaml]
....
api-key: <api-key>
....

=== Image filters

When searching for images, the driver queries the IBM Cloud VPC image API. Image filters help narrow down the list of cached images.

Each entry of `image-filters` enables one or more filters at once; all specified filters must match for an image to be included.

.Specification
[source,yaml]
....
image-filters:
    # Image architecture must match a regular expression
  - arch-regex: <pattern>

    # Image name must match a regular expression
    name-regex: <pattern>

    # Image creation date must match a regular expression
    creation-date-regex: <pattern>

    # Image must be younger than this many seconds
    max-age: <integer>

    # Image offer must match exactly
    offer: <string>

    # Image publisher must match exactly
    publisher: <string>

    # Image SKU must match exactly
    sku: <string>
....

=== Flavors

==== Default flavor

A default flavor is used when the guest request does not specify HW requirements that would narrow down the flavor selection:

.Specification
[source,yaml]
....
default-flavor: <flavor-name>
....

==== Flavor filtering

To limit which flavors are available, use a regular expression:

.Specification
[source,yaml]
....
flavor-regex: <pattern>
....

==== Use default flavor as fallback

When no suitable flavor matches the guest request, the default flavor can be used as a fallback:

.Specification
[source,yaml]
....
use-default-flavor-when-no-suitable: true  # default: true
....

==== Default root disk size

.Specification
[source,yaml]
....
default-root-disk-size: <integer>
....

=== Networking

==== Subnet

.Specification
[source,yaml]
....
subnet-id: <subnet-id>
....

=== SSH key

The driver requires a pre-existing SSH key pair for guest access:

.Specification
[source,yaml]
....
master-key-name: <key-name>
....

=== Region and zone

.Specification
[source,yaml]
....
default-region: <region>
zone: <zone>
....

.Example
[source,yaml]
....
default-region: "us-east"
zone: "us-east-2"
....

=== Resource group

.Specification
[source,yaml]
....
resource-group: <resource-group-name>
....

=== Expose public IP address

By default, Artemis exposes the private IP address of the instance. To expose the public IP instead:

.Specification
[source,yaml]
....
use-public-ip: true
....

=== Available guest logs

[NOTE]
====
For details on guest log support across all drivers, see <<drivers.adoc#_guest_logs,"Guest logs">>.
====

The IBM Cloud VPC driver does not currently support any guest logs.
